# Форма задач + Telegram бот

Готовый мини-сайт с публичной формой и административной панелью.

Что умеет проект:
- публичная форма без входа;
- хранение задач в SQLite;
- админка с полным списком, поиском, фильтрами, редактированием и удалением;
- Telegram-бот: краткая сводка по кнопкам и кнопки `Подробнее` по каждой записи;
- при необходимости мгновенное уведомление в Telegram о каждой новой записи.

## Страницы
- `/` — публичная форма для всех;
- `/admin/login` — вход в админку;
- `/admin` — панель администратора;
- `/admin/tasks` — все записи;
- `/admin/summary` — краткая сводка на сайте.

## Установка локально
```bash
pip install -r requirements.txt
uvicorn main:app --reload
```

Открыть в браузере:
```text
http://127.0.0.1:8000
```

## Логин по умолчанию
- `admin`
- по умолчанию в коде: `Qwerty00` (задаётся в `.env`)

Сразу поменяйте их через переменные окружения.

## Переменные окружения
Смотрите `.env.example`.

Минимально важные:
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `SECRET_KEY`
- `BOT_TOKEN`
- `BASE_URL` (для webhook; локально см. `TELEGRAM_POLLING`)
- `TELEGRAM_WEBHOOK_SECRET`
- `ADMIN_CHAT_ID` — только если `NOTIFY_ON_NEW=1` (куда слать пуш о новой заявке). Сводки в боте и кнопки доступны **любому**, кто написал боту.
- `NOTIFY_ON_NEW` — `0` по умолчанию: без авто-уведомлений в Telegram; таблица «Все записи» только по кнопке. Поставьте `1`, если нужно уведомление о каждой новой заявке.

## Подключение Telegram webhook
После деплоя задайте `BASE_URL`, `BOT_TOKEN`, `TELEGRAM_WEBHOOK_SECRET` (и при необходимости `ADMIN_CHAT_ID` для пушей о новых заявках).

Затем откройте в браузере такую ссылку:

```text
https://api.telegram.org/bot<BOT_TOKEN>/setWebhook?url=<BASE_URL>/telegram/webhook&secret_token=<TELEGRAM_WEBHOOK_SECRET>
```

Пример:

```text
https://api.telegram.org/bot123456:ABC/setWebhook?url=https://your-app.up.railway.app/telegram/webhook&secret_token=mysecret123
```

После этого напишите боту `/start`.

### Бот на своём компьютере (без публичного URL)
Telegram не доставляет апдейты на `http://127.0.0.1`. Пока нет HTTPS (Railway, ngrok и т.п.), в `.env` включите **`TELEGRAM_POLLING=1`** и перезапустите приложение. Бот перейдёт на long polling (при этом с бота снимется webhook — для продакшена снова вызовите `setWebhook` и поставьте **`TELEGRAM_POLLING=0`**).

## Кнопки бота
- `Все записи` — таблица по **всем** задачам (дата и время, инициатор, краткая формулировка); при большом числе строк сообщение может разбиться на несколько частей.
- `Сводка за сегодня`
- `Последние 10`
- `Новые за 7 дней`
- `Помощь`

Автоматически: каждый день в **16:30 по Москве** бот отправляет всем активным чатам полную таблицу «Все записи».

Под краткими сводками бот показывает кнопки `Подробнее #ID` по каждой записи.

## Railway
1. Загрузите проект в GitHub.
2. Создайте новый проект в Railway и подключите репозиторий.
3. После деплоя откройте `Settings -> Networking -> Generate Domain`.
4. Укажите этот домен в `BASE_URL`.
5. Добавьте переменные окружения из `.env.example`.
6. Для сохранения базы между деплоями подключите volume и задайте `DB_PATH=/data/tasks.db`.

## Что важно
- Без volume на Railway SQLite может не сохраняться после redeploy.
- Если Telegram не отвечает, проверьте `BOT_TOKEN`, `BASE_URL` / `TELEGRAM_POLLING` и установлен ли webhook.
